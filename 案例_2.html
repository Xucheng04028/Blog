该不该训练打字姿势

笑来说练习打字的最好方法和姑娘聊天，我也赞同这确实可以提升打字速度。但是，手势未必标准，这样长期打字会不会不太好？出于这样的考虑，我之前还特意查了些应用。

免费英文打字训练应用 Type Fu 已不再免费，而 Typist 是 Mac App Store 里少有的免费打字应用，但功能太过简单。所以之前最终以 TiPP10 作为打字姿势训练应用，因为它免费且强大，只是微微遗憾它不支持 Retina……

但到了现在，我已经长时间没有再练习打字姿势了，这也是我不想购买打字应用的原因，我预感到自己无法坚持，就像当初学习五笔输入法一样。

我必须得思考思考，也要查一些资料，所谓正确的打字姿势是否确实可以提高打字速度的上限…… 在这之前，我还是每天练习一会吧，不多就五分钟。

20181226 实际上不到一周就没练了，思考速度跟不上打字速度的感觉……

安卓手机如何通过数据线给电脑共享网络

一般而言绝大部分安卓手机都可以通过 USB 正常给 Windows 共享网络， 而 iPhone 共享网络给 Windows 只需安装 iTunes 并信任此电脑。如果发现安卓手机无法通过 USB 给 Windows 共享网络，那么就可以按照百度经验的 这篇教程 进行操作，但教程应该是不完全的转载，故仍有需补充的地方。

教程中提到在设备管理器中找到 Android Phone 以确定设备 ID 号，但也许找不到，比如我。那就注意在带黄色问号的其他设备中是否有 RNDIS，它的 PID 和 VID 就是手机设备 ID 号。然后在 这里 下载 tetherxp.inf 并按照教程中添加设备 ID 信息即可。

教程实际上就是安装 RNDIS 驱动，它是 Remote Network Driver Interface Specification（
远程网络驱动程序接口规范）的首字母缩写，实际上的作用为 TCP/IP over USB，也即视 USB 设备（如手机）为网卡，从而使 Windows 可以通过 USB 设备连接网络。TCP/IP 协议族，简称为 TCP/IP， 其名称源于其最早的两个核心协议 TCP（Transmission Control Protocol，传输控制协议）和 IP （Internet Protocol，网际协议），它规定了设备连接网络的方式以及数据如何传输的标准。

安卓手机共享网络给 Macintosh

由于 macOS 缺乏 RNDIS 驱动，安卓手机一般是无法与 macOS 通过数据线共享网络的。在 macOS 上安装 HoRNDIS 即可通过安卓手机共享网络。

不仅仅是 Markdown 写作工具

MarkEditor

支持平台：macOS & Windows
观看教程：点击这里

难得的是功能众多，仍不失优雅。情不自禁折服于 Z.R.E.Y 的审美水平，之前为 Farbox 倾心，现在 MarkEditor 亦如此。

在此，仅介绍我常用的功能。

  1. Markdown 写作模式
    这是 MarkEditor 的灵魂技能，开启 Markdown 写作模式以后，会自动将 Markdown 语法涉及的中文符号转换为英文符号，从此不必切换输入法。
  2. 工作目录
    连按两次 command 键可以快速调出设定好的文件目录。
  3. 同步到云端
    如七牛、Farbox等。设定七牛后，不仅具备了分享功能，还能自动将图片上传并转换为 Markdown 格式，写文章时，一气呵成，再也不必忧虑插入图片的问题，是运营静态博客的利器。
  4. 导出
    导出功能分为十几项，比较常用的导出为： Wiki 站点 & 电子书 & 微信公众号。Wiki 站点等都可以在局域网内或互联网分享。可以说，MarkEditor 解放了我们的双手。

优惠购买

购买 MarkEditor 授权时，可以输入已购买者的邮箱作为优惠码以享 七折或八折 优惠，每个邮箱总使用次数不超过五十次。

如果是 FarBox 用户：
输入 FarBox 中账户邮箱购买 MarkEditor 时，Pro 账户只需 RMB 1 购买 Pro 版，Senior 账户则可购买通用版。Standard 账户购买 MarkEditor，6 折优惠。如果 FarBox 中未绑定邮箱，需要先绑定邮箱。

亦可关注其微信公众号zreysoft或 官网，在特殊的日子会有更优惠的折扣。

本以为 MarkEditor 足以，但最近越来越依赖 Windows 平台，而 MarkEditor 在 Windows 下的高清屏显示效果令我倍感无奈，还好无意间邂逅的 Typora 使我顿时舒心颜。

Windows 版的高清屏下显示效果很糟？

MarkEditor 对 Windows 版下的高清屏支持是有限的，可以通过设置内调节日间、夜间的样式，里面涉及文字大小、尺寸相关的，尝试放大 2 倍或者其它合适的数值。
但是 ME 对 Windows 下的高清屏支持是有限的，请慎重购买。

Typora

支持平台：macOS & Windows & Linux

Typora 最大的特点是即时预览，不同于其他编辑器的双栏写作与预览，Typora 有 Word 的感觉，似乎有点偏离 Markdown 的本意。只有光标放置的位置才会显示出 Markdown 语法结构。

目前我常用两个功能：

  1. 插入链接
    选中需要插入链接的文字，按Ctrl + K即可自动插入剪贴板中的网址。
  2. 标题等级
    标题等级可以用快捷键进行切换。

学术期刊简介

期刊数据库

SCI，科学引文索引，Science Citation Index，美国科学情报研究所。主要收录数、理、化等自然科学的核心期刊，是国际公认最具权威的科技文献检索工具。SCI 期刊有两种主流的分区方法：JCR 分区与中国科学院分区。

  1. JCR 分区。将 SCI 期刊按照所属领域分为 177 个学科，然后将同一学科所有期刊根据该年的影响因子降序排序，分为 Q1（1% - 25%）、Q2（26% - 50%）、Q3（51% - 75%）、Q4（76% - 100%）；
  2. 中国科学院分区。根据该 SCI 期刊三年的平均影响因子，将期刊按所属领域分为 14 个大类与 174 个小类，然后将同一学科所有期刊根据该年的影响因子降序排序，分为一区（1% - 5%），二区（6% - 20%），三区（21% - 50%），四区（51% - 100%）。

EI，工程索引，The Engineering Index，美国工程信息公司。收录文献几乎涉及工程技术各个领域。

ISTP，科技会议索引，Index to Scientific & Technical Proceedings

CSCD，中国科学引文数据库，Chinese Science Citation Database

PKU，中文核心，北京大学图书馆中文核心期刊

CSTPCD，科技核心，中国科技论文与引文数据库，Chinese Scientific Papers and Citations Database

SCIE，科学引文索引扩展版，SCI Expanded

SSCI，社会科学引文索引，Social Sciences Citation Index，美国科学信息研究所。主要收录社会科学期刊。

CSSCI，中文社会科学引文索引，Chinese Social Sciences Citation Index，南京大学中国社会科学研究评价中心。

文献出版社

Elsevier，爱思唯尔（Elsevier）是世界上最大的医学与其他科学文献出版社之一，属于 RELX 集团旗下，总部位于阿姆斯特丹。

Windows 下验证哈希值

Windows 自带的文件校验和完整性验证程序（FCIV）可用于计算一个文件的 MD5 或 sha-1 哈希值。在命令行键入命令 FCIV-md5-sha1 path\filename.ext 以验证，比如 FCIV-md5-sha1 c:\windows\system32\shdocvw.dll。

iamCristYe：如果提示错误，建议使用 certutil -hashfile eg.ext SHA1 或 Get-FileHash eg.ext -Algorithm SHA1（后者需 PowerShell）安装相应工具。

解决 GitHub for macOS 的出错信息

很久之前在 segmentfault 回答的问题，但现在要停用此账号。

出错信息：

2014-11-19 20:41:30.130 GitHub for Mac Login[2595:326257] AskPass with arguments: (
    "/Applications/GitHub.app/Contents/MacOS/GitHub for Mac Login",
    "Username for 'https://github.com': "
)
2014-11-19 20:41:30.200 GitHub for Mac Login[2596:326280] AskPass with arguments: (
    "/Applications/GitHub.app/Contents/MacOS/GitHub for Mac Login",
    "Password for 'https://tcitry@github.com': "
)
fatal: unable to access 'https://github.com/tcitry/tcitry.github.io.git/': Empty reply from server
 (128)

解决方案

  1. 检查远程库地址
    进入客户端设置（Settings）检查远程库地址，应该是类似这样的地址：https://github.com/～.git
  2. 修改远程库地址
    按着此格式 git@github.com:～.git 修改远程库地址，然后点击 Update Remote
  3. 提交
    注意使用终端提交的同样修改提交地址即可。

问题分析

这应该不是 SSH KEY 的问题，应该也不是升级系统的原因。使用 HTTPS 连接会要求输入账户密码（这是加密连接，有些公司内网只支持此种连接方式）。但是，必须确保此时的公钥与私钥是正确对应的，尤其是经过诸多测试后。

  • 确认秘钥：

请看 官方帮助文档 提供的故障排除详细指南。如果问题仍在，那应该是没仔细观阅。

尤其注意 ssh -vT git@github.com 命令的运行结果：

Offering RSA public key: /Users/you/.ssh/id_rsa 说明调用的 KEY

Trying private key: /Users/you/.ssh/id_rsa 说明 KEY 不存在

注意 GitHub客户端会自动生成 KEY（github_rsa & github_rsa.pub）并自动添加公钥。所以不必再生成 KEY 了。

即使再生成也没有影响，两者默认命名是不同的，不会覆盖。

  • 郁闷

但是我的的客户端昨天还好好的，今天就得改下远程库地址连接方式才能提交。好诡异。寻找原因中。愿与诸位共讨论。

关于维基百科的编辑

MediaWiki 的前世今生在 其词条 中已有详细的描述。了解维基百科，我强烈推荐仔细阅读 Semantic MediaWiki 中的一系列文章。

不论是 Markdown 还是 WikiPedia 语法最终都是被解析为 HTML 语法，所以它们都可以与 HTML 语法混用，但也有 一些限制。

关于如何编辑维基百科和其语法可在 帮助：编辑页面 得到充足的信息。当然也最好看看 维基百科：格式手册，这有助于编写出符合维基风格的词条。

签名

最初我一直不懂签名是什么意思，写下自己的用户名？后来发现留下签名一般就是留下四个波浪号 ~~~~，它会被自动解析为 签名 + 时间；而三个波浪号 ~~~ 只被解析为签名；五个波浪号 ~~~~~ 只被解析为时间。

解析方式

有时我们不想输入的文本被解析和格式化，那么可以使用 <pre> </pre> 标签；如果不想被解析但想被格式化，可用 <nowiki> </nowiki> 标签。

对应关系

维基百科使用 MediaWiki 语法，而我这只小白用户只熟悉 Markdown 语法。在编辑维基百科时总是要查询或者切换可视化编辑器，在这里总结下 MediaWiki 语法。

## 二级标题 == 二级标题 ==
### 三级标题 === 三级标题 ===

1. 有序列表 # 有序列表
2. 无序列表 # 有序列表

- 无序列表 * 无序列表

[文字](http://www.test.com)
[http://www.test.com 文字]

![图片](http://www.test.com)
[[File:Wiki.png]]

MediaWiki 语法中代码使用 HTML 标签表示：<code>代码</code>
