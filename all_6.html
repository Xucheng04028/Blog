全平台 Outlook 指南

iCloud 曾经是我的主力平台，通讯录、备注、提醒事项、日历等都由 iCloud 同步。但是 iCloud 最大的问题就是封闭，无法做到跨平台同样的体验。从前我的打算是只使用 Apple 设备，但现在来看这并不太可能了。因为微软的二合一设备对我就有很大的吸引力，安卓手机也偶尔使用。

Outlook 是一款很优秀的产品，最重要的特性是一个应用可以解决所有平台的问题。Outlook 的五大功能：邮件、日历、人脉、任务、便签，使我们对学习和工作都应对自如。只是 Outlook 似乎没有得到微软的重视，有许多细节体验让人心碎。本文的目的是介绍作者的 Outlook 全平台使用经验。

Outlook for macOS 的联系人、任务、便签使用起来并不顺心，为此我同时设置了 Exchange 账户，选择同步邮件、日历、联系人、任务、便签。实际上完全也可以只使用 Exchange，我只是追求 Windows 与 macOS 的无缝衔接。Outlook for macOS 的联系人对中文姓名的优化不太好，而 macOS 的通讯录则优化颇佳；Outlook 的联系人中有许多选项无法更改名称，也有一些不需要的功能，而 macOS 的通讯录功能则简洁且全面。Outlook 的任务和便签仅在新建环节上就让人崩溃，而且新建的便签组会被当作网页版 Outlook 邮箱的文件夹，所以我强烈建议通过 Exchange 使用 macOS 自带的应用。

Outlook for Android 的功能跟 for iOS 一样，但体验差得很远。而且许多安卓手机的 Exchange 无法同步任务和便签。所以在 Android 上我只建议使用一个被称作最佳的 Outlook 客户端：Nine，在谷歌商店中可免费试用，解锁价格大概十刀，在其他应用商店中也许免费提供。Nine 支持同步邮件、日历、人脉、任务和便签，虽有些复杂，但还是比较优美的。

Outlook for iOS 并不会同步任务和便签，所以我只使用了 Exchange 同步邮件、日历、人脉、任务与便签。

Outlook for Windows 就无须介绍了。

总结一下：

  1. macOS 上建议用 Exchange 同步，也可以搭配 Outlook 客户端使用；
  2. iOS 上建议只用 Exchange 同步；
  3. Android 上建议使用三方 Outlook 客户端 Nine；
  4. Windows 根据系统版本考虑，建议使用 Outlook 客户端。

不仅仅是 Markdown 写作工具

MarkEditor

支持平台：macOS & Windows
观看教程：点击这里

难得的是功能众多，仍不失优雅。情不自禁折服于 Z.R.E.Y 的审美水平，之前为 Farbox 倾心，现在 MarkEditor 亦如此。

在此，仅介绍我常用的功能。

  1. Markdown 写作模式 这是 MarkEditor 的灵魂技能，开启 Markdown 写作模式以后，会自动将 Markdown 语法涉及的中文符号转换为英文符号，从此不必切换输入法。
  2. 工作目录 连按两次 command 键可以快速调出设定好的文件目录。
  3. 同步到云端 如七牛、Farbox等。设定七牛后，不仅具备了分享功能，还能自动将图片上传并转换为 Markdown 格式，写文章时，一气呵成，再也不必忧虑插入图片的问题，是运营静态博客的利器。
  4. 导出 导出功能分为十几项，比较常用的导出为： Wiki 站点 & 电子书 & 微信公众号。Wiki 站点等都可以在局域网内或互联网分享。可以说，MarkEditor 解放了我们的双手。

优惠购买

购买 MarkEditor 授权时，可以输入已购买者的邮箱作为优惠码以享 七折或八折 优惠，每个邮箱总使用次数不超过五十次。

如果是 FarBox 用户：
输入 FarBox 中账户邮箱购买 MarkEditor 时，Pro 账户只需 RMB 1 购买 Pro 版，Senior 账户则可购买通用版。Standard 账户购买 MarkEditor，6 折优惠。如果 FarBox 中未绑定邮箱，需要先绑定邮箱。

亦可关注其微信公众号zreysoft或 官网，在特殊的日子会有更优惠的折扣。

本以为 MarkEditor 足以，但最近越来越依赖 Windows 平台，而 MarkEditor 在 Windows 下的高清屏显示效果令我倍感无奈，还好无意间邂逅的 Typora 使我顿时舒心颜。

Windows 版的高清屏下显示效果很糟？

MarkEditor 对 Windows 版下的高清屏支持是有限的，可以通过设置内调节日间、夜间的样式，里面涉及文字大小、尺寸相关的，尝试放大 2 倍或者其它合适的数值。
但是 ME 对 Windows 下的高清屏支持是有限的，请慎重购买。

Typora

支持平台：macOS & Windows & Linux

Typora 最大的特点是即时预览，不同于其他编辑器的双栏写作与预览，Typora 有 Word 的感觉，似乎有点偏离 Markdown 的本意。只有光标放置的位置才会显示出 Markdown 语法结构。

目前我常用两个功能：

  1. 插入链接 选中需要插入链接的文字，按Ctrl + K即可自动插入剪贴板中的网址。
  2. 标题等级 标题等级可以用快捷键进行切换。

不再用百度的所有服务

我以为“百毒”是恶意的称呼，我错了。

  1. 百度钱包 最敏感的信息。但是并不能删除。
  2. 百度网盘 这无疑是最艰难的删除决定，我原来费心整理的资料都在里面。但这些本科的资料、备份的应用和系统镜像，似乎我也不大能用到。资源占有者也并非我的目标，资源掌握者才是。清空回收站。
  3. 百度站长 删除网站信息。联系人手机号也要更改。
  4. 百度文库 曾经在文库下载了许多有用的资料，甚至还有同学借我的号下载。虽然有点不舍财富值，但似乎也并没什么大用。部分信息没法删除，但不影响。
  5. 百度百科 什么都无法删除，但影响不大。
  6. 百度知道 删除回答需要财富值，部分提问无法删除。遗留了许多无法删除的信息，但还好吧。
  7. 百度学术 求助信息无法删除，已删除收藏及订阅信息。
  8. 百度贴吧 删除数十条贴子。而回复太多，也无法批量删除，就搁置吧。回收站无法清理。
  9. 百度外卖 订单信息无法删除，已删除地址。
  10. 百度音乐 全部删除。
  11. 百度图片 删除我的图片。
  12. 百度脑图 删除文件。
  13. 解除三方关联，解绑手机

学术期刊简介

期刊数据库

SCI

科学引文索引，Science Citation Index，美国科学情报研究所。主要收录数、理、化等自然科学的核心期刊，是国际公认最具权威的科技文献检索工具。SCI 期刊有两种主流的分区方法：JCR 分区与中国科学院分区。

  • JCR 分区 将 SCI 期刊按照所属领域分为 177 个学科，然后将同一学科所有期刊根据该年的影响因子降序排序，分为 Q1（1% - 25%）、Q2（26% - 50%）、Q3（51% - 75%）、Q4（76% - 100%）。
  • 中国科学院分区 根据该 SCI 期刊三年的平均影响因子，将期刊按所属领域分为 14 个大类与 174 个小类，然后将同一学科所有期刊根据该年的影响因子降序排序，分为一区（1% - 5%），二区（6% - 20%），三区（21% - 50%），四区（51% - 100%）。

EI

工程索引，The Engineering Index，美国工程信息公司。收录文献几乎涉及工程技术各个领域。

ISTP

科技会议索引，Index to Scientific & Technical Proceedings,

CSCD

中国科学引文数据库，Chinese Science Citation Database

PKU

中文核心，北京大学图书馆中文核心期刊

CSTPCD

科技核心，中国科技论文与引文数据库，Chinese Scientific Papers and Citations Database

SCIE

科学引文索引扩展版，SCI Expanded

SSCI

社会科学引文索引，Social Sciences Citation Index，美国科学信息研究所。主要收录社会科学期刊。

CSSCI

中文社会科学引文索引，Chinese Social Sciences Citation Index，南京大学中国社会科学研究评价中心。

文献出版社

Elsevier

爱思唯尔（Elsevier）是世界上最大的医学与其他科学文献出版社之一，属于 RELX 集团旗下，总部位于阿姆斯特丹。
